# Pitfalls

## Common Failure Modes

- Failure: Jumping to solutions too early
  Cause: Background understanding is still weak
  Fix: Expand facts, glossary, and open-question mapping before planning implementation

- Failure: Mixing facts with guesses
  Cause: Notes are written too loosely
  Fix: Label sections explicitly as facts, assumptions, open questions, and recommendations

- Failure: Over-documenting low-value details
  Cause: Everything gets captured at the same level
  Fix: Keep `SKILL.md` and project summaries concise; move detail into targeted reference notes

- Failure: Losing track of why decisions were made
  Cause: Decisions are kept only in chat history
  Fix: Record significant choices in `tasks/decisions.md`

## Safety Notes
- Avoid broad rewrites when the project is still exploratory
- Prefer additive documentation updates
- Keep project boundaries clear when multiple explorations exist
