# Workflow

## Goal
Provide a repeatable process for turning an unclear or research-heavy project into a structured exploratory project with usable documentation.

## Standard Flow
1. Define the project theme in one or two sentences.
2. Collect core background concepts.
3. Create a glossary of important terms.
4. Record known facts and unknowns.
5. Map the most important questions.
6. Turn the current understanding into a short action plan.

## What to Produce First
- `context/project-overview.md`
- `context/glossary.md`
- `context/research-notes.md`
- `tasks/active.md`

## Decision Points
- If the project is mostly educational, focus on concept maps and glossary quality.
- If the project is likely to become implementation work, also record architecture candidates and decision logs early.
- If uncertainty remains high, avoid overcommitting to a solution too early.

## Validation
- Can a new collaborator understand the project from the docs?
- Are facts and assumptions clearly separated?
- Are the next steps specific and actionable?
