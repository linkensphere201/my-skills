# Document Structure

## Recommended Project Files
- `README.md`: short project summary for humans
- `AGENTS.md`: project-specific rules for Codex
- `context/project-overview.md`: goals, scope, status
- `context/glossary.md`: key terms
- `context/research-notes.md`: facts, assumptions, open questions, findings
- `context/architecture.md`: optional technical structure and options
- `tasks/active.md`: current work
- `tasks/backlog.md`: pending ideas
- `tasks/decisions.md`: major decisions and rationale
- `logs/worklog.md`: chronological notes

## Writing Guidance
- Put stable knowledge in `context/`
- Put changing work state in `tasks/`
- Put chronological narrative in `logs/`
- Keep repeated information in one place only

## Early-Stage Priority
When the project is still fuzzy, prioritize:
1. `project-overview.md`
2. `glossary.md`
3. `research-notes.md`
4. `active.md`
